 \hypertarget{doc_help_kdb-set_md_autotoc_md3054}{}\doxysection{SYNOPSIS}\label{doc_help_kdb-set_md_autotoc_md3054}
{\ttfamily kdb set $<$key name$>$ $<$value$>$}

Where {\ttfamily key name} is the name of the key you wish to set the value of (or create) and {\ttfamily value} is the value you would like to set the key to.\hypertarget{doc_help_kdb-set_md_autotoc_md3055}{}\doxysection{DESCRIPTION}\label{doc_help_kdb-set_md_autotoc_md3055}
This command allows the user to set the value of an individual key. If a cascading key is used that does not resolve to an existing key, the operation is aborted as it is ambiguous.\hypertarget{doc_help_kdb-set_md_autotoc_md3056}{}\doxysection{EMPTY VALUES}\label{doc_help_kdb-set_md_autotoc_md3056}
To set a key to an empty value, {\ttfamily \char`\"{}\char`\"{}} should be passed for the {\ttfamily value} argument.\hypertarget{doc_help_kdb-set_md_autotoc_md3057}{}\doxysection{NEGATIVE VALUES}\label{doc_help_kdb-set_md_autotoc_md3057}
To set a key to a negative value, {\ttfamily -\/-\/} has to be used to stop option processing. (see example below)\hypertarget{doc_help_kdb-set_md_autotoc_md3058}{}\doxysection{OPTIONS}\label{doc_help_kdb-set_md_autotoc_md3058}

\begin{DoxyItemize}
\item {\ttfamily -\/H}, {\ttfamily -\/-\/help}\+: Show the man page.
\item {\ttfamily -\/V}, {\ttfamily -\/-\/version}\+: Print version info.
\item {\ttfamily -\/p}, {\ttfamily -\/-\/profile $<$profile$>$}\+: Use a different kdb profile.
\item {\ttfamily -\/C}, {\ttfamily -\/-\/color $<$when$>$}\+: Print never/auto(default)/always colored output.
\item {\ttfamily -\/q}, {\ttfamily -\/-\/quiet}\+: Suppress non-\/error messages.
\item {\ttfamily -\/-\/}\+: Do not process any following arguments starting with {\ttfamily -\/} as options.
\item {\ttfamily -\/v}, {\ttfamily -\/-\/verbose}\+: Explain what is happening. Prints additional information in case of errors/warnings.
\item {\ttfamily -\/d}, {\ttfamily -\/-\/debug}\+: Give debug information. Prints additional debug information in case of errors/warnings.
\item {\ttfamily -\/f}, {\ttfamily -\/-\/force}\+: Do not perform a cascading lookup if the key provided has a namespace. For example, this bypasses validation specified in the spec\+: namespace for the given key.
\end{DoxyItemize}\hypertarget{doc_help_kdb-set_md_autotoc_md3059}{}\doxysection{RETURN VALUES}\label{doc_help_kdb-set_md_autotoc_md3059}

\begin{DoxyItemize}
\item 0\+: No errors.
\item 11\+: Not a valid keyname.
\item 12\+: Setting a non-\/existing key without a namespace is not possible.
