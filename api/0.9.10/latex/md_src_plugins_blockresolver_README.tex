The {\ttfamily blockresolver} can be used to only resolve a tagged block inside a configuration file.\hypertarget{md_src_plugins_blockresolver_README_autotoc_md64}{}\doxysection{Installation}\label{md_src_plugins_blockresolver_README_autotoc_md64}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The package is called {\ttfamily libelektra5-\/experimental}.\hypertarget{md_src_plugins_blockresolver_README_autotoc_md65}{}\doxysubsection{Implementation Details}\label{md_src_plugins_blockresolver_README_autotoc_md65}
{\ttfamily blockresolver} extracts the requested block from the configurations file and writes it into a temporary file. Afterwards Elektra will only work on the temporary file until kdb\+Set is called. On kdb\+Set the contents of the temporary file will be merged with parts outside of the requested block from the original file.\hypertarget{md_src_plugins_blockresolver_README_autotoc_md66}{}\doxysection{Usage}\label{md_src_plugins_blockresolver_README_autotoc_md66}
\begin{DoxyVerb}`kdb mount -R blockresolver /path/to/my/file /mountpoint -c identifier="identifier-tag"`
\end{DoxyVerb}
 where {\ttfamily identifier} specifies the tag {\ttfamily blockresolver} will search for in the configuration file.

A block consists of 2 parts\+:


\begin{DoxyItemize}
\item beginning\+: the identifier suffixed with {\ttfamily start}
\item end\+: the identifier suffixed with {\ttfamily stop}
\end{DoxyItemize}\hypertarget{md_src_plugins_blockresolver_README_autotoc_md67}{}\doxysection{Limitations}\label{md_src_plugins_blockresolver_README_autotoc_md67}
Currently the identifier must be unique.\hypertarget{md_src_plugins_blockresolver_README_autotoc_md68}{}\doxysection{Example}\label{md_src_plugins_blockresolver_README_autotoc_md68}
