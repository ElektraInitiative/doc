This plugin is a storage plugin allowing Elektra to read and write XML formatted files. It uses the legacy Elektra 0.\+7 XML format.

This plugin can be used for migration of Key Databases from 0.\+7 -\/$>$ 0.\+8. It should not be used otherwise.\hypertarget{md_src_plugins_xmltool_README_autotoc_md636}{}\doxysection{Installation}\label{md_src_plugins_xmltool_README_autotoc_md636}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The package is called {\ttfamily libelektra5-\/xmltool}.\hypertarget{md_src_plugins_xmltool_README_autotoc_md637}{}\doxysection{Dependencies}\label{md_src_plugins_xmltool_README_autotoc_md637}

\begin{DoxyItemize}
\item {\ttfamily libxml2-\/dev}
\end{DoxyItemize}\hypertarget{md_src_plugins_xmltool_README_autotoc_md638}{}\doxysection{Restrictions}\label{md_src_plugins_xmltool_README_autotoc_md638}

\begin{DoxyItemize}
\item only supports metadata as defined in Elektra 0.\+7
\item null and empty values are not distinguished
\item exported relative to first key found, not to parent key (ks\+Get\+Common\+Parent\+Name)
\item error messages vague (no difference between error opening file and validation errors)
\end{DoxyItemize}\hypertarget{md_src_plugins_xmltool_README_autotoc_md639}{}\doxysection{Examples}\label{md_src_plugins_xmltool_README_autotoc_md639}
After you have upgraded Elektra, you can import XML files from Elektra 0.\+7\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb import system:/ xmltool < system.xml}
\DoxyCodeLine{kdb import user:/ xmltool < user.xml}

\end{DoxyCode}


Or you can also mount an XML file using {\ttfamily xmltool} (not recommended)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount /etc/example.xml system:/example xmltool}

\end{DoxyCode}
