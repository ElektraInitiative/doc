{\ttfamily kdb meta-\/get $<$key name$>$ $<$metaname$>$}~\newline


Where {\ttfamily key name} is the name of the key and {\ttfamily metaname} is the name of the metakey the user would like to access.\hypertarget{doc_help_kdb-meta-get_md_autotoc_md2455}{}\doxysection{DESCRIPTION}\label{doc_help_kdb-meta-get_md_autotoc_md2455}
This command is used to print the value of a metakey. A metakey is information stored in a key which describes that key.

The handling of cascading {\ttfamily key name} does not differ to {\ttfamily kdb get}. Make sure to use the namespace {\ttfamily spec}, if you want metadata from there.\hypertarget{doc_help_kdb-meta-get_md_autotoc_md2456}{}\doxysection{RETURN VALUES}\label{doc_help_kdb-meta-get_md_autotoc_md2456}
This command will return the following values as an exit status\+:~\newline



\begin{DoxyItemize}
\item 0\+: No errors.
\item 1\+: Key not found. (Invalid {\ttfamily path})
\item 2\+: Meta key not found. (Invalid {\ttfamily metaname}).
\end{DoxyItemize}\hypertarget{doc_help_kdb-meta-get_md_autotoc_md2457}{}\doxysection{OPTIONS}\label{doc_help_kdb-meta-get_md_autotoc_md2457}

\begin{DoxyItemize}
\item {\ttfamily -\/H}, {\ttfamily -\/-\/help}\+: Show the man page.
\item {\ttfamily -\/V}, {\ttfamily -\/-\/version}\+: Print version info.
\item {\ttfamily -\/p}, {\ttfamily -\/-\/profile $<$profile$>$}\+: Use a different kdb profile.
