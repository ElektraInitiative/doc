 \hypertarget{doc_decisions_0b_rejected_separate_key_name_md_autotoc_md1914}{}\doxysection{Problem}\label{doc_decisions_0b_rejected_separate_key_name_md_autotoc_md1914}
Often a {\ttfamily Key} argument is used when you just need a key name. This is because with a {\ttfamily Key} we know the name is valid and we get an unescaped name. Using a {\ttfamily Key} here makes the API a bit confusing.

There could be a richer API for manipulating key names without relying on the escaped name (e.\+g. concatenating two full key names). With the current situation, all these functions would need to be part of the API for a key. Adding such functions to the key API is certainly not minimal.\hypertarget{doc_decisions_0b_rejected_separate_key_name_md_autotoc_md1915}{}\doxysection{Constraints}\label{doc_decisions_0b_rejected_separate_key_name_md_autotoc_md1915}
