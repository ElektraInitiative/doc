\hypertarget{doc_help_kdb-record-export_md_autotoc_md3065}{}\doxysection{DESCRIPTION}\label{doc_help_kdb-record-export_md_autotoc_md3065}
This command exports the recorded changes into the specified output format. Keys are exported to {\ttfamily stdout} in whichever format is specified.\hypertarget{doc_help_kdb-record-export_md_autotoc_md3066}{}\doxysection{USAGE}\label{doc_help_kdb-record-export_md_autotoc_md3066}
Where {\ttfamily source} is the path of the key(s) you want to export. Additionally, the user can specify a format to use by passing it as the option argument {\ttfamily format}. The {\ttfamily format} attribute relies on Elektraâ€™s plugin system to export the keys in the desired format. The parameter {\ttfamily format} must be the name of a valid storage plugin.

Both {\ttfamily source} and {\ttfamily format} are optional parameters. By default, {\ttfamily source} is {\ttfamily /} and {\ttfamily format} is {\ttfamily ansible}. If you only specify a single argument, we will whether it is a key or the format.\hypertarget{doc_help_kdb-record-export_md_autotoc_md3067}{}\doxysection{OPTIONS}\label{doc_help_kdb-record-export_md_autotoc_md3067}
