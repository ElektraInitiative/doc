This plugin is an Elektra to Ruby bridge and makes is possible to implement Elektra plugins with Ruby. This plugin requires the {\ttfamily ruby} binding.\hypertarget{md_src_plugins_ruby_README_autotoc_md517}{}\doxysection{Installation}\label{md_src_plugins_ruby_README_autotoc_md517}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The package is called {\ttfamily libelektra5-\/ruby}.\hypertarget{md_src_plugins_ruby_README_autotoc_md518}{}\doxysection{Configuration Options}\label{md_src_plugins_ruby_README_autotoc_md518}
The plugin has the following configuration options\+:


\begin{DoxyItemize}
\item {\ttfamily script}\+: path to the Ruby plugin to use (mandatory)
\end{DoxyItemize}\hypertarget{md_src_plugins_ruby_README_autotoc_md519}{}\doxysection{Usage}\label{md_src_plugins_ruby_README_autotoc_md519}
Mount a configuration file using this plugin, which specifying the concrete Ruby Elektra plugin\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount /.ssh/authorized\_keys user:/ssh/authorized\_keys ruby script=<path to elektra}
\DoxyCodeLine{source>/src/plugins/ruby/ruby/ssh\_authorized\_keys.rb}
\DoxyCodeLine{kdb ls user:/ssh/authorized\_keys}

\end{DoxyCode}

\hypertarget{md_src_plugins_ruby_README_autotoc_md521}{}\doxysection{Known Issues}\label{md_src_plugins_ruby_README_autotoc_md521}

\begin{DoxyItemize}
\item The plugin might cause a SIGSEGV crash on application shutdown. This is a result of missing Ruby\+VM de initialization, which is silently ignored as we can\textquotesingle{}t determine if the Ruby VM is still needed or not (libelektra might be used by an Ruby application).
\item The plugin does not work correctly with Ruby version managers like {\ttfamily rbenv} or {\ttfamily rvm} and is intended to be used with the Ruby ecosystem installed in system context. 
\end{DoxyItemize}
