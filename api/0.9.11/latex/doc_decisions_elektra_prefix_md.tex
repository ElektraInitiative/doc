 \hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1788}{}\doxysection{Problem}\label{doc_decisions_elektra_prefix_md_autotoc_md1788}
Some names, such as kdb\+Open, key\+New are so generic that other libraries might also use them.\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1789}{}\doxysection{Constraints}\label{doc_decisions_elektra_prefix_md_autotoc_md1789}

\begin{DoxyItemize}
\item In C such libraries, containing the same external identifier, cannot be used together.
\item The \href{https://www.open-std.org/jtc1/sc22/wg14/}{\texttt{ C99 standard, section 5.\+2.\+4.\+1}} gives following limit\+: 31 significant initial characters in an external identifier (each universal character name specifying a short identifier of 0000FFFF or less is considered 6 characters, each universal character name specifying a short identifier of 00010000 or more is considered 10 characters, and each extended source character is considered the same number of characters as the corresponding universal character name, if any)
\end{DoxyItemize}\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1790}{}\doxysection{Assumptions}\label{doc_decisions_elektra_prefix_md_autotoc_md1790}
\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1791}{}\doxysection{Considered Alternatives}\label{doc_decisions_elektra_prefix_md_autotoc_md1791}

\begin{DoxyItemize}
\item leave it as is
\item use macros to have short names for actually longer external identifiers
\end{DoxyItemize}\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1792}{}\doxysection{Decision}\label{doc_decisions_elektra_prefix_md_autotoc_md1792}

\begin{DoxyItemize}
\item Ensure all functions start with {\ttfamily elektra}.
\item Ensure all macros and constants start with {\ttfamily ELEKTRA\+\_\+}.
\end{DoxyItemize}\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1793}{}\doxysection{Rationale}\label{doc_decisions_elektra_prefix_md_autotoc_md1793}

\begin{DoxyItemize}
\item This makes it clear which functions come from Elektra.
\item Unifies all function names within Elektra (including core).
\item Avoids collisions with other libs.
\end{DoxyItemize}\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1794}{}\doxysection{Implications}\label{doc_decisions_elektra_prefix_md_autotoc_md1794}

\begin{DoxyItemize}
\item Changes in basically every application and tool, but this is automated with a refactoring tool @kodebach writes.
\item To be 100\% C99 compatible we cannot introduce identifiers which the same 31 character prefix (e.\+g. {\ttfamily ELEKTRA\+\_\+\+WARNING\+\_\+\+VALIDATION\+\_\+\+SYNTACTIC\+\_\+\+LEXER} and {\ttfamily ELEKTRA\+\_\+\+WARNING\+\_\+\+VALIDATION\+\_\+\+SYNTACTIC\+\_\+\+PARSER} would be a problem). Otherwise, it is not guaranteed that the code can be compiled with every C99 compiler.
\end{DoxyItemize}\hypertarget{doc_decisions_elektra_prefix_md_autotoc_md1795}{}\doxysection{Related Decisions}\label{doc_decisions_elektra_prefix_md_autotoc_md1795}
