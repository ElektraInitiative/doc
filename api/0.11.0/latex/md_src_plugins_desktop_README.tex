The plugin is informational and mainly be used to provide context for other configuration. See elektrify-\/getenv.\hypertarget{md_src_plugins_desktop_README_autotoc_md178}{}\doxysection{Installation}\label{md_src_plugins_desktop_README_autotoc_md178}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The package is called {\ttfamily libelektra5-\/extra}.\hypertarget{md_src_plugins_desktop_README_autotoc_md179}{}\doxysection{Usage}\label{md_src_plugins_desktop_README_autotoc_md179}
To mount the plugin please use\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo kdb mount -\/-\/resolver noresolver none system:/info/desktop desktop}

\end{DoxyCode}


or it is already included if you already mounted the info plugins with\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount-\/info}

\end{DoxyCode}


Then you can get desktop information via\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb get system:/info/desktop}

\end{DoxyCode}


You either get a {\itshape lower-\/case} string (supported desktops see below) or no key if no desktop was detected.\hypertarget{md_src_plugins_desktop_README_autotoc_md180}{}\doxysection{Supported Desktops}\label{md_src_plugins_desktop_README_autotoc_md180}
Currently supported desktops are\+:


\begin{DoxyItemize}
\item GNOME
\item KDE
\item TDE
\item Unity
\item XDG conformant desktops ({\ttfamily XDG\+\_\+\+CURRENT\+\_\+\+DESKTOP})
\end{DoxyItemize}

Currently the detection relies on environment variables, which will not work in setuid or otherwise secured binaries. Please open a bug report if the detection does not work for you\+: \href{https://issues.libelektra.org}{\texttt{ https\+://issues.\+libelektra.\+org}}\hypertarget{md_src_plugins_desktop_README_autotoc_md181}{}\doxysection{Unmount the plugin}\label{md_src_plugins_desktop_README_autotoc_md181}
To unmount the plugin you can run


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo kdb umount system:/info/desktop}

\end{DoxyCode}
