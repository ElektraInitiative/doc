 In Elektra string-\/keys are preferred. Nevertheless, binary keys, i.\+e. strings with null-\/characters embedded are possible.\hypertarget{doc_help_elektra-values_md_autotoc_md2308}{}\doxysection{ABSENT KEYS}\label{doc_help_elektra-values_md_autotoc_md2308}
Sometimes a key does not exist at all. When using \mbox{\hyperlink{group__keyset_ga60f1ddcf23272f2b29b90e92ebe9b56f}{ks\+Lookup()}} you will get a null pointer. In the C++ Binding the null pointer is wrapped in a key. Thus this is also sometimes called {\bfseries{NULL KEYS}}.\hypertarget{doc_help_elektra-values_md_autotoc_md2309}{}\doxysection{NULL VALUES}\label{doc_help_elektra-values_md_autotoc_md2309}
Null values are binary values without content. They are the only keys that have the size 0.

Null values are always binary values.\hypertarget{doc_help_elektra-values_md_autotoc_md2310}{}\doxysection{EMPTY VALUES}\label{doc_help_elektra-values_md_autotoc_md2310}
Empty values point to a string that only contains a null byte.

Empty values are possible for both string and binary values.
