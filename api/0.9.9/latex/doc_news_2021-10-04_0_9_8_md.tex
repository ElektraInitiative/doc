Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.8-release}{\texttt{ on our website}}.

You can try out the latest Elektra release using our docker image \href{https://hub.docker.com/r/elektra/elektra}{\texttt{ elektra/elektra}}. This is the quickest way to get started with Elektra without compiling and other obstacles, simply run\+:


\begin{DoxyCode}{0}

\DoxyCodeLine{docker run -\/it elektra/elektra}

We have created a version of \href{https://github.com/jonls/redshift/}{\texttt{ Redshift}} that uses Elektra for configuration management!

We removed and refactored Redshift\textquotesingle{}s code for loading configuration files, parsing CLI options and validating configuration to use Elektra. Redshift with Elektra has about 700 fewer lines of code (-\/16\%) and is a great example of what Elektra is all about\+: More applications with less code for configuration management!

To test it, take a look at our PR \href{https://github.com/jonls/redshift/pull/837}{\texttt{ Refactor to use Elektra}} and follow the instructions provided in \href{https://github.com/qwepoizt/redshift/blob/improve-config/elektrify/CONTRIBUTING.md}{\texttt{ CONTRIBUTING.\+md}}.\hypertarget{doc_news_2021-10-04_0_9_8_md_autotoc_md3535}{}\doxysubsubsection{Benefits of Redshift using Elektra}\label{doc_news_2021-10-04_0_9_8_md_autotoc_md3535}
Refactoring Redshift to use Elektra brings the following benefits\+:


\begin{DoxyItemize}
\item Fewer lines of code\+: reduction of $\sim$700 LOC or 16\% (measured across all {\ttfamily $\ast$.c} and {\ttfamily $\ast$.h} files exclusive of files automatically generated by Elektra).
\item Adding new configuration settings is easier and takes less time\+: Validation and parsing of setting values (from configuration file and CLI options) is handled by Elektra -\/ no custom code required!
\item Clean separation of application code and the specification of supported configuration settings (including defaults and validation rules).
\item Automatic generation of CLI help text.
\end{DoxyItemize}\hypertarget{doc_news_2021-10-04_0_9_8_md_autotoc_md3536}{}\doxysubsection{HL API improvements}\label{doc_news_2021-10-04_0_9_8_md_autotoc_md3536}
Redshift with Elektra uses the \href{https://www.libelektra.org/tutorials/high-level-api}{\texttt{ high-\/level API}}.

We have made a large number of improvements to the \href{https://www.libelektra.org/tutorials/high-level-api}{\texttt{ high-\/level API}} in the course of refactoring Redshift. The highlights are\+:


\begin{DoxyItemize}
\item Improved detection of differences in specification between an application\textquotesingle{}s compilation and runtime.
\item Improved validation of CLI options.
\item Early detection of errors in specification files.
\item Updated and improved documentation and tutorials.
\item Various other bugfixes.
\end{DoxyItemize}

More improvements and details are explained in later sections of these release notes.
