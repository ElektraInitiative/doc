\item {\bfseries{Scope\+:}} {\ttfamily spec}

\item {\bfseries{Brief\+:}} This use case introduces an array specification for a part of the dockerd configuration file (daemon.\+json).
\end{DoxyItemize}\hypertarget{doc_usecases_specification_array-specification_md_autotoc_md4715}{}\doxysection{Scenarios}\label{doc_usecases_specification_array-specification_md_autotoc_md4715}

\begin{DoxyItemize}
\item {\bfseries{Precondition\+:}} The Dev-\/\+Ops Engineer has a working setup for {\ttfamily docker} for and the daemon {\ttfamily dockerd}.
\item {\bfseries{Main success scenario\+:}}
\begin{DoxyItemize}
\item The Dev-\/\+Ops Engineer wants to write a specification for the {\ttfamily storage options} in the {\ttfamily dockerd} configuration file.
\item The array specification configuration key is {\ttfamily storage/opts/\#}.
\item The configuration uses {\ttfamily array/min} and {\ttfamily description} as metakeys.
\item The array is defined by the wildcard character {\ttfamily \#}.
\item The keys are all stored for the {\ttfamily spec} namespace.
\end{DoxyItemize}
\item {\bfseries{Alternative scenario\+:}}
\begin{DoxyItemize}
\item Define the storage type with an underline specification.
\item The array specification configuration key is {\ttfamily storage/opts/\+\_\+}.
\item The configuration uses {\ttfamily array/min} and {\ttfamily description} as metakeys.
\end{DoxyItemize}
\item {\bfseries{Error scenario\+:}}
\begin{DoxyItemize}
\item Wrong metakeys are used (yielded as error to the user).
\end{DoxyItemize}
\item {\bfseries{Postcondition\+:}} The keys are all stored for the {\ttfamily spec} namespace.
\item {\bfseries{Non-\/functional Constraints\+:}} None.
\end{DoxyItemize}\hypertarget{doc_usecases_specification_array-specification_md_autotoc_md4716}{}\doxysection{Example}\label{doc_usecases_specification_array-specification_md_autotoc_md4716}
