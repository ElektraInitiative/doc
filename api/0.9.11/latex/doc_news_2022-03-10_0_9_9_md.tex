Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.9-release}{\texttt{ on our website}}.

You can try out the latest Elektra release using our docker image \href{https://hub.docker.com/r/elektra/elektra}{\texttt{ elektra/elektra}}. This is the quickest way to get started with Elektra without compiling and other obstacles, simply run\+:

As you will read in this release notes, a massive amount of bugs were fixed within this release. In the upcoming term there will be a course about \href{https://tiss.tuwien.ac.at/course/courseDetails.xhtml?courseNr=194030&semester=2022S}{\texttt{ configuration management}} in which Elektra will also be used and improved upon.\hypertarget{doc_news_2022-03-10_0_9_9_md_autotoc_md3619}{}\doxysubsection{Java Plugins}\label{doc_news_2022-03-10_0_9_9_md_autotoc_md3619}
The version of the {\ttfamily process} plugin, makes it much easier to implement plugins in Java. You can now call an implementation {\ttfamily \mbox{\hyperlink{interfaceorg_1_1libelektra_1_1Plugin}{org.\+libelektra.\+Plugin}}} via {\ttfamily process} and the {\ttfamily org.\+libelektra.\+process.\+Plugin\+Process} class.
