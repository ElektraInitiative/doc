This caching plugin stores keysets from previous {\ttfamily \mbox{\hyperlink{group__kdb_ga28e385fd9cb7ccfe0b2f1ed2f62453a1}{kdb\+Get()}}} calls to improve performance when reading configuration files.\hypertarget{md_src_plugins_cache_README_autotoc_md67}{}\doxysection{Usage}\label{md_src_plugins_cache_README_autotoc_md67}
The cache plugin is compiled and enabled on compatible systems by default. No actions are needed to enable it.\hypertarget{md_src_plugins_cache_README_autotoc_md68}{}\doxysection{Dependencies}\label{md_src_plugins_cache_README_autotoc_md68}
POSIX compliant system (including XSI extensions). The plugin is only compiled if the plugins {\ttfamily resolver} and {\ttfamily mmapstorage} are also available.\hypertarget{md_src_plugins_cache_README_autotoc_md69}{}\doxysection{Location of Cache}\label{md_src_plugins_cache_README_autotoc_md69}
The cache files are located in the user\textquotesingle{}s home directory below {\ttfamily $\sim$/.cache/elektra/} and shall not be altered, otherwise the behavior is undefined. If {\ttfamily XDG\+\_\+\+CACHE\+\_\+\+HOME} is set, the cache files are located below {\ttfamily \$\+XDG\+\_\+\+CACHE\+\_\+\+HOME/elektra}.\hypertarget{md_src_plugins_cache_README_autotoc_md70}{}\doxysection{Configuration of Cache}\label{md_src_plugins_cache_README_autotoc_md70}
Use the tool {\ttfamily kdb cache} to enable, disable or clear the cache.\hypertarget{md_src_plugins_cache_README_autotoc_md71}{}\doxysection{Limitations}\label{md_src_plugins_cache_README_autotoc_md71}
Incompatible with storage plugins, which do not always produce the same keyset on any invocation concerning the same configuration file. A notable example here is the {\ttfamily ini} plugin (see issue \#2592).
