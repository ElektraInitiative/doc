We shifted our \href{https://git.libelektra.org/blob/master/doc/GOALS.md}{\texttt{ goals}} a bit\+: We want to prefer simplicity to flexibility. Not because we do no like flexibility, but because we think we achieved enough of it. Currently (and in future) you can use Elektra\+:


\begin{DoxyItemize}
\item obviously as primitive key-\/value storage
\item with specifications and dozens of plugins driven by it
\item with code generation
\item ...
\end{DoxyItemize}

But we cut flexibility regarding\+:


\begin{DoxyItemize}
\item namespaces are only useful for configuration (not for arbitrary key-\/value)
\item the semantics of \href{https://git.libelektra.org/blob/master/doc/METADATA.ini}{\texttt{ metadata}}
\item mounting functionality
\item error code meanings are fixed, if a resolver detects a conflict, our defined error must be used
\item of course ABI, API
\end{DoxyItemize}\hypertarget{doc_news_2015-11-19_0_8_14_md_autotoc_md3216}{}\doxysection{Qt-\/gui 0.\+0.\+9}\label{doc_news_2015-11-19_0_8_14_md_autotoc_md3216}

ENABLE\+\_\+\+CXX11 does not exist anymore, it is always on. We do not care about 199711L compilers anymore, which makes development easier, without losing any actually used platform.

Python and Lua plugins are enabled now in {\ttfamily -\/DPLUGINS=ALL}.

Python3 plugin was renamed to python.\hypertarget{doc_news_2015-11-19_0_8_14_md_autotoc_md3219}{}\doxysection{Lua Plugin}\label{doc_news_2015-11-19_0_8_14_md_autotoc_md3219}
Manuel Mausz add a lightweight alternative to the python plugin\+: \href{https://master.libelektra.org/src/plugins/lua/}{\texttt{ the lua plugin}}. In a similar way, someone can write scripts, which are executed on every access to the \href{https://master.libelektra.org/doc/help/elektra-glossary.md}{\texttt{ key database}}.

To mount a lua based filter, you can use\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount file.ini /lua ini lua script=/path/to/lua/lua\_filter.lua}

\end{DoxyCode}


Even though it works well, it is classified as technical preview.

This plugin allows you to check and even calculate keys from other keys using an polish prefix notation. It supports the typical operations {\ttfamily + -\/ / $\ast$} and {\ttfamily $<$, $<$=, ==, !=, =$>$, $>$, \+:=}. To mount, check and calculate values, one would use\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount mathcheck.dump /example/mathcheck dump mathcheck}
\DoxyCodeLine{kdb setmeta user/example/mathcheck/k check/math "{}== + a b"{}}
\DoxyCodeLine{kdb setmeta user/example/mathcheck/k check/math "{}:= + a b"{}}

\end{DoxyCode}


For details \href{https://master.libelektra.org/src/plugins/mathcheck/}{\texttt{ see the documentation}}.

Thanks to Thomas Waser for this important plugin!\hypertarget{doc_news_2015-11-19_0_8_14_md_autotoc_md3223}{}\doxysection{List Plugin}\label{doc_news_2015-11-19_0_8_14_md_autotoc_md3223}
Currently, Elektra has some limitations on how many plugins can be added to certain \href{https://master.libelektra.org/doc/dev/plugins-ordering.md}{\texttt{ placement}}. Because of the rapidly growing number of plugins, some combinations are not possible anymore.

This plugin tackles the issue, by delegating the work to an arbitrary number of subplugins. As a bonus, it works lazily and thus might avoid the loading of some plugins all together.

Thanks to Thomas Waser for this plugin!\hypertarget{doc_news_2015-11-19_0_8_14_md_autotoc_md3224}{}\doxysection{Conditionals}\label{doc_news_2015-11-19_0_8_14_md_autotoc_md3224}
Brings {\ttfamily if} inside Elektra. It lets you check if some keys have the values they should have.


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount conditionals.dump /tmount/conditionals conditionals dump}
\DoxyCodeLine{kdb set user/tmount/conditionals/fkey 3.0}
\DoxyCodeLine{kdb set user/tmount/conditionals/hkey hello}
\DoxyCodeLine{kdb setmeta user/tmount/conditionals/key check/condition "{}(hkey == 'hello') ? (fkey == '3.0')"{} \# success}
\DoxyCodeLine{kdb setmeta user/tmount/conditionals/key check/condition "{}(hkey == 'hello') ? (fkey == '5.0')"{} \# fail}

\end{DoxyCode}


For details \href{https://master.libelektra.org/src/plugins/conditionals/}{\texttt{ see the documentation}}.

Again, thanks to Thomas Waser for this plugin!\hypertarget{doc_news_2015-11-19_0_8_14_md_autotoc_md3225}{}\doxysection{Csvstorage Plugin}\label{doc_news_2015-11-19_0_8_14_md_autotoc_md3225}
