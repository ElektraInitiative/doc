This plugin parses {\ttfamily passwd} files, e.\+g. {\ttfamily /etc/passwd}.\hypertarget{md_src_plugins_passwd_README_autotoc_md395}{}\doxysection{Installation}\label{md_src_plugins_passwd_README_autotoc_md395}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The package is called {\ttfamily libelektra5-\/experimental}.\hypertarget{md_src_plugins_passwd_README_autotoc_md396}{}\doxysection{Implementation Details}\label{md_src_plugins_passwd_README_autotoc_md396}
The non-\/\+POSIX function {\ttfamily fgetpwent} (GNU\+\_\+\+SOURCE) will be used to read the file supplied by the resolver. As a fallback we implemented our own version based on musls {\ttfamily fgetpwent}.

For writing {\ttfamily putpwent} (GNU\+\_\+\+SOURCE) will be used. If it is not available the plugin will write straight to the config file.\hypertarget{md_src_plugins_passwd_README_autotoc_md397}{}\doxysection{Requirements}\label{md_src_plugins_passwd_README_autotoc_md397}
For the plugin to be build at least {\ttfamily POSIX\+\_\+\+C\+\_\+\+SOURCE $>$= 200809L} compatibility is required.\hypertarget{md_src_plugins_passwd_README_autotoc_md398}{}\doxysection{Configuration}\label{md_src_plugins_passwd_README_autotoc_md398}
If the config key {\ttfamily index} is set to {\ttfamily name} passwd entries will be sorted by name, if not set or set to {\ttfamily uid} passwd entries will be sorted by uid\hypertarget{md_src_plugins_passwd_README_autotoc_md399}{}\doxysection{Fields}\label{md_src_plugins_passwd_README_autotoc_md399}

\begin{DoxyItemize}
\item {\ttfamily gecos} contains the full name of the account
\item {\ttfamily gid} contains the accounts primary group id
\item {\ttfamily home} contains the path to the accounts home directory
\item {\ttfamily shell} contains the accounts default shell
\item {\ttfamily uid} contains the accounts uid
\item {\ttfamily name} contains the account name
