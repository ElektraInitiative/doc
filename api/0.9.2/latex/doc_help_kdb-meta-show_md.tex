{\ttfamily kdb meta-\/show $<$key name$>$}~\newline


Where {\ttfamily key name} is the name of the key the user would like to access.\hypertarget{doc_help_kdb-meta-show_md_autotoc_md1768}{}\doxysection{D\+E\+S\+C\+R\+I\+P\+T\+I\+ON}\label{doc_help_kdb-meta-show_md_autotoc_md1768}
This command is used to print all metakeys and its values for a given key. A metakey is information stored in a key which describes that key.

The handling of cascading {\ttfamily key name} does not differ to {\ttfamily kdb get}. Make sure to use the namespace {\ttfamily spec}, if you want metadata from there.\hypertarget{doc_help_kdb-meta-show_md_autotoc_md1769}{}\doxysection{R\+E\+T\+U\+R\+N V\+A\+L\+U\+ES}\label{doc_help_kdb-meta-show_md_autotoc_md1769}
This command will return the following values as an exit status\+:~\newline



\begin{DoxyItemize}
\item 0\+: No errors.
\item 1\+: Key not found. (Invalid {\ttfamily path})
