{\ttfamily kdb sget $<$key name$>$ $<$default-\/value$>$}

Where {\ttfamily key name} is the name of the key to retrieve and {\ttfamily default-\/value} is the value that should be printed if no value can be retrieved.\hypertarget{doc_help_kdb-sget_md_autotoc_md2551}{}\doxysection{DESCRIPTION}\label{doc_help_kdb-sget_md_autotoc_md2551}
This command is used to retrieve the value of a key from within a script. When using the kdb tool in a script, the user should use the {\ttfamily sget} command in place of the kdb-\/get(1) command. The kdb-\/get(1) command should not be used in scripts because it may return an error instead of printing a value in certain circumstances. The {\ttfamily sget} command guarantees that a value will be printed (unless the user passes faulty arguments). This command will either print the value of the key it retrieves or a default value that the user specifies.\hypertarget{doc_help_kdb-sget_md_autotoc_md2552}{}\doxysection{OPTIONS}\label{doc_help_kdb-sget_md_autotoc_md2552}

\item {\ttfamily -\/V}, {\ttfamily -\/-\/version}\+: Print version info.
\item {\ttfamily -\/p}, {\ttfamily -\/-\/profile $<$profile$>$}\+: Use a different kdb profile.
\item {\ttfamily -\/C}, {\ttfamily -\/-\/color $<$when$>$}\+: Print never/auto(default)/always colored output.

 {\ttfamily kdb sget user\+:/example/key 0}
