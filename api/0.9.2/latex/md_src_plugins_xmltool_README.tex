This plugin is a storage plugin allowing Elektra to read and write X\+ML formatted files. It uses the {\ttfamily libelektratools} 0.\+7 X\+ML format.

This plugin can be used for migration of Key Databases from 0.\+7 -\/$>$ 0.\+8. It should not be used otherwise.\hypertarget{md_src_plugins_xmltool_README_autotoc_md725}{}\doxysection{Dependencies}\label{md_src_plugins_xmltool_README_autotoc_md725}

\begin{DoxyItemize}
\item {\ttfamily libxml2-\/dev}
\end{DoxyItemize}\hypertarget{md_src_plugins_xmltool_README_autotoc_md726}{}\doxysection{Restrictions}\label{md_src_plugins_xmltool_README_autotoc_md726}

\begin{DoxyItemize}
\item only supports metadata as defined in Elektra 0.\+7
\item null and empty values are not distinguished
\item exported relative to first key found, not to parent key (ks\+Get\+Common\+Parent\+Name)
\item error messages vague (no difference between error opening file and validation errors)
\end{DoxyItemize}\hypertarget{md_src_plugins_xmltool_README_autotoc_md727}{}\doxysection{Examples}\label{md_src_plugins_xmltool_README_autotoc_md727}
After you have upgraded Elektra, you can import X\+ML files from Elektra 0.\+7\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb import system xmltool < system.xml}
\DoxyCodeLine{kdb import user xmltool < user.xml}
\end{DoxyCode}


Or you can also mount an X\+ML file using {\ttfamily xmltool} (not recommended)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb mount /etc/example.xml system/example xmltool}
\end{DoxyCode}
