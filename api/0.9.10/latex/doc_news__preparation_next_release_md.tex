We are proud to release Elektra 0.\+9.$<$$<$\+VERSION$>$$>$.\hypertarget{doc_news__preparation_next_release_md_autotoc_md3623}{}\doxysection{What is Elektra?}\label{doc_news__preparation_next_release_md_autotoc_md3623}
Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.<<VERSION>>-release}{\texttt{ on our website}}.

You can try out the latest Elektra release using our docker image \href{https://hub.docker.com/r/elektra/elektra}{\texttt{ elektra/elektra}}. This is the quickest way to get started with Elektra without compiling and other obstacles, simply run\+:

We created a new Binding for Kotlin with convenience functions and various utilities. There is also the possibility to convert Key\+Sets to Kotlin data classes or collections and back. Read the Kotlin Readme for more information.

A big thanks to \+\_\+(@\+Gratla and @mandoway)\+\_\+ for this beautiful work.\hypertarget{doc_news__preparation_next_release_md_autotoc_md3626}{}\doxysubsection{Remove Internal Iterators}\label{doc_news__preparation_next_release_md_autotoc_md3626}
In Elektra there are currently two different ways to iterate over Key\+Sets. The so-\/called \char`\"{}internal\char`\"{} iterator is, however, inferior and creates several problems, e.\+g. it was a side effect to be considered for every function call that involved a Key\+Set.

With this release, we started removing the internal iterators by removing {\ttfamily key\+Rewind\+Meta}, {\ttfamily key\+Current\+Meta}, {\ttfamily ks\+Head}, and {\ttfamily ks\+Tail} functions. The external iterators are now the way to go, see \mbox{\hyperlink{doc_dev_iterators_md}{Iterators}}.

A huge thanks to \+\_\+(Florian Lindner @flo91)\+\_\+ for doing this thankless cleanup task.\hypertarget{doc_news__preparation_next_release_md_autotoc_md3627}{}\doxysection{Plugins}\label{doc_news__preparation_next_release_md_autotoc_md3627}
