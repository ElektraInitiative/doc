This filter plugin checks if a Key is a valid domain name, i.\+e. if the name can be resolved into an IPv4 address. The validation takes place whenever the {\ttfamily kdb set} method is called.\hypertarget{md_src_plugins_python_python_dns_README_autotoc_md420}{}\doxysection{Usage}\label{md_src_plugins_python_python_dns_README_autotoc_md420}
The python plugin requires the configuration parameter {\bfseries{script}} holding the file path to the python script. The mount command looks like


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo kdb mount file.ini /python ni python script=/path/to/dns\_plugin.py}

\end{DoxyCode}


For the plugin to actually check a certain key, the {\ttfamily check/dns} metakey must be set.


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo kdb meta-\/set spec:/python/my\_hostname check/dns ''}

\end{DoxyCode}


Now each time the key is set, the filter plugin validates the given value.


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb set user:/python/my\_hostname www.libelektra.org}

\end{DoxyCode}
