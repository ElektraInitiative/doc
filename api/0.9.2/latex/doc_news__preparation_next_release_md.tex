We are proud to release Elektra 0.\+9.\+2.

This release focuses on bugfixes and stability improvements, but also brings some new features. With the 0.\+9.\+x series of releases we shift our focus to K\+DE and G\+N\+O\+ME integration.\hypertarget{doc_news__preparation_next_release_md_autotoc_md2455}{}\doxysection{What is Elektra?}\label{doc_news__preparation_next_release_md_autotoc_md2455}
Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.<<VERSION>>-release}{\texttt{ on our website}}\hypertarget{doc_news__preparation_next_release_md_autotoc_md2456}{}\doxysection{Highlights}\label{doc_news__preparation_next_release_md_autotoc_md2456}

We created a \href{https://github.com/ElektraInitiative/kconfig}{\texttt{ fork}} of \href{https://kde.org/}{\texttt{ K\+DE\textquotesingle{}s}} {\ttfamily K\+Config} configuration system and patched it to use libelektra. We have done some initial testing and replaced the {\ttfamily K\+Config} library for \href{https://kate-editor.org/}{\texttt{ Kate}} and \href{https://www.kdevelop.org/}{\texttt{ K\+Develop}} successfully. Additionally, a new Elektra plugin called {\ttfamily kconfig} was added, which can read and write K\+DE\textquotesingle{}s kconfig ini files to help migrate to Elektra\textquotesingle{}s {\ttfamily K\+Config} fork. \+\_\+(\+Dardan Haxhimustafa)\+\_\+ and \+\_\+(\+Felix Resch)\+\_\+\hypertarget{doc_news__preparation_next_release_md_autotoc_md2458}{}\doxysubsection{G\+N\+O\+M\+E Integration}\label{doc_news__preparation_next_release_md_autotoc_md2458}
We continued work on Elektra\textquotesingle{}s bindings for \href{https://developer.gnome.org/gio/stable/GSettings.html}{\texttt{ G\+N\+O\+ME G\+Settings}}. Our implementation should be able to replace the widely used \href{https://wiki.gnome.org/Projects/dconf}{\texttt{ dconf}} backend. Elektra\textquotesingle{}s {\ttfamily gsettings} bindings are not yet ready for production use, but they are already able to replace dconf for a complete G\+N\+O\+ME session without problems. We are still lacking proper dbus integration for change notifications. \+\_\+(\+Gabriel Rauter)\+\_\+ and \+\_\+(Mihael Pranjić)\+\_\+\hypertarget{doc_news__preparation_next_release_md_autotoc_md2459}{}\doxysubsection{$<$tt$>$elektrad$<$/tt$>$ rewritten in Go}\label{doc_news__preparation_next_release_md_autotoc_md2459}
\href{https://www.libelektra.org/tools/elektrad}{\texttt{ elektrad}} provides an H\+T\+TP A\+PI to access Elektra remotely. {\ttfamily elektrad} is now completely rewritten in Go, which drastically improves the performance by leveraging the new \href{https://github.com/ElektraInitiative/go-elektra/}{\texttt{ go-\/elektra}} bindings instead of calling the {\ttfamily kdb} command-\/line tool on every request. The new Elektrad creates a session per user to reuse the same kdb handle for better performance. \+\_\+(\+Raphael Gruber)\+\_\+\hypertarget{doc_news__preparation_next_release_md_autotoc_md2460}{}\doxysection{Plugins}\label{doc_news__preparation_next_release_md_autotoc_md2460}
