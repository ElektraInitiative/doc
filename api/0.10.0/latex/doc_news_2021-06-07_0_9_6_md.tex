We are proud to release Elektra 0.\+9.\+6, the 7th release in preparation for Elektra 1.\+0.\hypertarget{doc_news_2021-06-07_0_9_6_md_autotoc_md3859}{}\doxysection{What is Elektra?}\label{doc_news_2021-06-07_0_9_6_md_autotoc_md3859}
Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.6-release}{\texttt{ on our website}}.

You can try out the latest Elektra release using our docker image \href{https://hub.docker.com/r/elektra/elektra}{\texttt{ elektra/elektra}}. This is the quickest way to get started with Elektra without compiling and other obstacles, simply run\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{docker pull elektra/elektra}
\DoxyCodeLine{docker run -\/it elektra/elektra}

\end{DoxyCode}

The JNI plugin was encountering a double free on open. This has been fixed in conjunction with an update to JNA binding release mechanism. The previously disabled JNI test have been fixed and enabled.

For how to write plugins, please refer to \mbox{\hyperlink{doc_tutorials_java-plugins_md}{java-\/plugins.md}} as well as the \mbox{\hyperlink{md_src_plugins_jni_README_src_plugins_jni_README_md}{JNI plugin}} and JNA binding documentation.
