We are proud to release Elektra 0.\+9.\+11.\hypertarget{doc_news_2022-10-05_0_9_11_md_autotoc_md4047}{}\doxysection{What is Elektra?}\label{doc_news_2022-10-05_0_9_11_md_autotoc_md4047}
Elektra serves as a universal and secure framework to access configuration settings in a global, hierarchical key database. For more information, visit \href{https://libelektra.org}{\texttt{ https\+://libelektra.\+org}}.

You can also read the news \href{https://www.libelektra.org/news/0.9.11-release}{\texttt{ on our website}}.

You can try out the latest Elektra release using our Docker image \href{https://hub.docker.com/r/elektra/elektra}{\texttt{ elektra/elektra}}. This is the quickest way to get started with Elektra without compiling and other obstacles, simply run\+:

\DoxyCodeLine{docker run -\/it elektra/elektra}

In a separated branch we rewrote the whole backend system. With this new backend system, backends are now also plugins, allowing backends also to be non-\/file-\/based, e.\+g., using databases. The main purpose of this release is to give a last stable release before master gets disrupted with a huge changeset. The next release is not to be expected in this year.

A huge thanks to \+\_\+(Klemens Böswirth @kodebach)\+\_\+, \+\_\+(Maximilian Irlinger @atmaxinger)\+\_\+ for the many changes in the branch. Thanks to \+\_\+(Richard Stöckl @\+Eiskasten)\+\_\+ for testing.\hypertarget{doc_news_2022-10-05_0_9_11_md_autotoc_md4050}{}\doxysubsection{Olimex}\label{doc_news_2022-10-05_0_9_11_md_autotoc_md4050}
Elektra is used for \mbox{\hyperlink{doc_WHO_md}{server, desktop and embedded}}. With this release, we strengthen our embedded mainstay, specifically in \href{https://freedomdefined.org/OSHW}{\texttt{ open source hardware OSHW}}. We developed a major application running on OSHW \href{https://www.olimex.com/Products/OLinuXino/open-source-hardware}{\texttt{ Olimex}} boards. The application is called \href{https://opensesame.libelektra.org}{\texttt{ opensesame}}. It is heavily relying on Elektra and \href{https://github.com/ElektraInitiative/ansible-libelektra}{\texttt{ ansible-\/libelektra}}.

In the initial release \href{https://opensesame.libelektra.org}{\texttt{ opensesame}} already allows\+:


\begin{DoxyItemize}
\item \mbox{[}x\mbox{]} opening (garage) doors via a novel PIN entry method\+: you can press and release buttons in any sequence
\item \mbox{[}x\mbox{]} switching on entry lights
\item \mbox{[}x\mbox{]} ringing doorbells
\item \mbox{[}x\mbox{]} detection of fire
\item \mbox{[}x\mbox{]} report events to Nextcloud chats (English and German)
\end{DoxyItemize}

To give a smoother experience when running such an application we will develop Ansible scripts to customize the Olimex base images. They will allow changing the language, time zone, static network configuration etc.

Olimex likes this idea and will send us an A20 board. A big thanks to \href{https://www.olimex.com}{\texttt{ Olimex}}.\hypertarget{doc_news_2022-10-05_0_9_11_md_autotoc_md4051}{}\doxysubsection{Improving Elektra in FLOSS course}\label{doc_news_2022-10-05_0_9_11_md_autotoc_md4051}
Also in the \href{https://tiss.tuwien.ac.at/course/courseDetails.xhtml?courseNr=194114&semester=2022W}{\texttt{ upcoming term}} Elektra will be object of study how FLOSS initiatives work. Students will make improvements in Elektra as part of their homework, teamwork and, if chosen, also for their project.
