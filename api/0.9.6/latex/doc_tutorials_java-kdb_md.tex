When programming in Java it is possible to access the key database, changing values of existing keys or adding new ones and a few other things. It is also possible to write plugins for Elektra in Java but we will focus on using the Java binding in this tutorial.\hypertarget{doc_tutorials_java-kdb_md_autotoc_md3546}{}\doxysection{First Steps}\label{doc_tutorials_java-kdb_md_autotoc_md3546}
In order to use {\ttfamily kdb} you need to include the dependency in your project. Here you can find a detailed tutorial on how to do that.

After that you can start loading a {\ttfamily KDB} object as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var key = Key.createNameless();}
\DoxyCodeLine{\textcolor{keywordflow}{try} (KDB \mbox{\hyperlink{namespacekdb}{kdb}} = KDB.open(key)) \{}
\DoxyCodeLine{    \textcolor{comment}{// code to manipulate keys}}
\DoxyCodeLine{\} \textcolor{keywordflow}{catch} (KDB.KDBException e) \{}
\DoxyCodeLine{    e.printStackTrace();}
\DoxyCodeLine{\} \textcolor{keywordflow}{finally} \{}
\DoxyCodeLine{    key.release(); \textcolor{comment}{// optional clean-\/up}}
\DoxyCodeLine{\}}

\end{DoxyCode}


Note that {\ttfamily KDB} implements {\ttfamily Auto\+Closable} which allows \href{https://docs.oracle.com/javase/tutorial/essential/exceptions/tryResourceClose.html}{\texttt{ {\ttfamily try-\/with-\/resouces}}}.

The key being passend to {\ttfamily KDB\+::open} is used to store warnings and error information. The special key name {\ttfamily Key.\+KEY\+\_\+\+LOCAL\+\_\+\+NAME} is used to create a key only used locally for transferring this information. If an error occurs, it will be mapped to the appropriate specialization of {\ttfamily KDBException}.

The following code is equivalent and preferred, if you do not want to reuse an existing key for transferring warnings and error information\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordflow}{try} (KDB \mbox{\hyperlink{namespacekdb}{kdb}} = KDB.open()) \{}
\DoxyCodeLine{    \textcolor{comment}{// code to manipulate keys}}
\DoxyCodeLine{\} \textcolor{keywordflow}{catch} (KDB.KDBException e) \{}
\DoxyCodeLine{    e.printStackTrace();}
\DoxyCodeLine{    e.releaseErrorKey(); \textcolor{comment}{// optional clean-\/up}}
\DoxyCodeLine{\}}

\end{DoxyCode}
\hypertarget{doc_tutorials_java-kdb_md_autotoc_md3547}{}\doxysection{A word about releasing native ressources}\label{doc_tutorials_java-kdb_md_autotoc_md3547}
There are 3 kinds of native resources having to be cleaned up properly to prevent memory leaks\+: {\ttfamily Key\+Set}, {\ttfamily Key} and {\ttfamily KDB} or rather their backing native key set, key or KDB session.

Fortunately the only resource you are strictly required to release is {\ttfamily KDB} via either {\ttfamily KDB\+::close} or {\ttfamily try-\/with-\/resouces}. For {\ttfamily Key\+Set} and {\ttfamily Key} the garbage collector cleans them up automatically using a \href{https://docs.oracle.com/en/java/javase/11/docs/api/java.base/java/lang/ref/Cleaner.html}{\texttt{ {\ttfamily Cleaner}}}. Nonetheless, it might be a good idea to clean up these native resources as soon as they are no longer needed, since we do not have any control of when or if the garbage collector cleans up after us.\hypertarget{doc_tutorials_java-kdb_md_autotoc_md3548}{}\doxysection{Fetching keys}\label{doc_tutorials_java-kdb_md_autotoc_md3548}
First let\textquotesingle{}s retrieve a key which is already part of the key database. The first thing we need to do is to create a {\ttfamily Key\+Set} your keys are going to be stored in\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var keySet = KeySet.create();}

\end{DoxyCode}


Now we load all keys located below a sepcific parent key\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var parentKey = Key.create(\textcolor{stringliteral}{"{}user:/"{}});}
\DoxyCodeLine{\mbox{\hyperlink{namespacekdb}{kdb}}.get(keySet, parentKey);}

\end{DoxyCode}


Now we can simply fetch the desired key\textquotesingle{}s value as follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{String value = keySet.lookup(\textcolor{stringliteral}{"{}user:/my/presaved/key"{}}).map(Key::getStringAndRelease).orElseThrow();}

\end{DoxyCode}


So for example if you had executed the command below via shell, before starting the application\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{kdb set user:/my/presaved/key it\_works!}

\end{DoxyCode}


{\ttfamily value} would equals {\ttfamily it\+\_\+works!}.

Afterward, we may clean-\/up no longer needed native resources\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{parentKey.release();}
\DoxyCodeLine{keySet.release();}

\end{DoxyCode}
\hypertarget{doc_tutorials_java-kdb_md_autotoc_md3549}{}\doxysection{Saving Keys}\label{doc_tutorials_java-kdb_md_autotoc_md3549}
Next let\textquotesingle{}s save a new key to the key database. Again, first we need need to create an empty {\ttfamily Key\+Set}. We also {\bfseries{need to fetch}} all keys for the namespace before we will be able to save a new key.


\begin{DoxyCode}{0}
\DoxyCodeLine{var keyNamespace = Key.create(\textcolor{stringliteral}{"{}user:/"{}});                 \textcolor{comment}{// create key representing the namespace to fetch}}
\DoxyCodeLine{var keySet = \mbox{\hyperlink{namespacekdb}{kdb}}.get(keyNamespace);                      \textcolor{comment}{// fetch all keys for the namespace into a new key set}}
\DoxyCodeLine{var keyToStore = Key.create(\textcolor{stringliteral}{"{}user:/somekey"{}}, \textcolor{stringliteral}{"{}myValue"{}}); \textcolor{comment}{// create key with value to store}}
\DoxyCodeLine{keySet.append(keyToStore);}
\DoxyCodeLine{\mbox{\hyperlink{namespacekdb}{kdb}}.set(keySet, keyToStore);}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// optional clean-\/up}}
\DoxyCodeLine{keySet.release();}
\DoxyCodeLine{keyNamespace.release();}
\DoxyCodeLine{keyToStore.release();}

\end{DoxyCode}


If you try to save a key without fetching it beforehand, a {\ttfamily KDBException} will be thrown, telling you to call get before set.

The {\itshape user} namespace is accessible without special rights, but if you try to write to {\itshape system} you will need to have root privileges. Take a look at \mbox{\hyperlink{doc_TESTING_md}{TESTING.md}} to see how to access the system namespace as non-\/root user. This should only be done in testing environments though as it is not intended for productive systems.\hypertarget{doc_tutorials_java-kdb_md_autotoc_md3550}{}\doxysection{Examples}\label{doc_tutorials_java-kdb_md_autotoc_md3550}

First we create a new {\ttfamily KDB} session and fetch all keys for the desired namespace, in this example the whole {\ttfamily user} namespace. Since all all keys are put in the passed {\ttfamily key\+Set} variable we can then iterate through it. The {\ttfamily at(int)} method return a new {\ttfamily Key} object for the native key with the corresponding position within the {\ttfamily key\+Set}.
