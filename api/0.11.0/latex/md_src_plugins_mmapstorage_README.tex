This is a high performance storage plugin that supports full Elektra semantics.\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md357}{}\doxysection{Format}\label{md_src_plugins_mmapstorage_README_autotoc_md357}
The storage format uses Elektra\textquotesingle{}s in-\/memory data layout and employs the {\ttfamily mmap()} system call to read/write data. The format is not portable across different architectures/platforms. The format can be seen as a memory dump of a keyset. Therefore, the files must not be edited by hand. Files written by mmapstorage are not intended to be human-\/readable.\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md358}{}\doxysection{Usage}\label{md_src_plugins_mmapstorage_README_autotoc_md358}

\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md359}{}\doxysection{Compiling}\label{md_src_plugins_mmapstorage_README_autotoc_md359}
The mmapstorage has two compilation variants\+:


\begin{DoxyEnumerate}
\item mmapstorage
\item mmapstorage\+\_\+crc
\end{DoxyEnumerate}

The {\ttfamily mmapstorage} will always be compiled on a supported system (see \href{\#dependencies}{\texttt{ Dependencies}}). When zlib is available, we will additionally compile the {\ttfamily mmapstorage\+\_\+crc} variant. The first variant does not do a CRC32 checksum of the critical data, while the second variant always checks the CRC32 checksum for additional security.\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md360}{}\doxysection{Installation}\label{md_src_plugins_mmapstorage_README_autotoc_md360}
See \mbox{\hyperlink{doc_INSTALL_md}{installation}}. The {\ttfamily mmapstorage} variant is part of the {\ttfamily libelektra5} package and the {\ttfamily mmapstorage\+\_\+crc} is part of the {\ttfamily libelektra5-\/extra} package.\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md361}{}\doxysection{Dependencies}\label{md_src_plugins_mmapstorage_README_autotoc_md361}
POSIX compliant system (including XSI extensions).

Additionally, zlib is needed for the {\ttfamily mmapstorage\+\_\+crc} compilation variant\+: {\ttfamily zlib1g-\/dev} or {\ttfamily zlib-\/devel}.\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md362}{}\doxysection{Examples}\label{md_src_plugins_mmapstorage_README_autotoc_md362}

\hypertarget{md_src_plugins_mmapstorage_README_autotoc_md363}{}\doxysection{Limitations}\label{md_src_plugins_mmapstorage_README_autotoc_md363}
Mapped files shall not be altered, otherwise the behavior is undefined.

The {\ttfamily mmap()} system call only supports regular files and so does the mmapstorage plugin with one notable exception\+: The plugin detects when it is called with the files {\ttfamily /dev/stdin} and {\ttfamily /dev/stdout} and makes an internal copy. This makes the plugin compatible with {\ttfamily kdb import} and {\ttfamily kdb export}.
